# Security Templates

This folder contains templates for compliance documentation. These are **TEMPLATES ONLY** - do not store actual signed agreements or confidential records here.

## Available Templates

### 1. [Acceptable Use Policy](./ACCEPTABLE_USE_POLICY.md)
Complete AUP defining appropriate use of company systems, data, and resources.

**Use this for:**
- Employee onboarding
- Annual policy acknowledgment
- Contractor/vendor agreements

**How to use:**
1. Review and customize for your company
2. Have employees read and acknowledge
3. Store signed acknowledgments in compliance folder

---

### 2. [Employee Confidentiality Agreement](./EMPLOYEE_CONFIDENTIALITY_AGREEMENT.md)
Combined NDA and FERPA compliance agreement template.

**Use this for:**
- All new employees before granting data access
- Contractors/consultants with data access
- Annual re-acknowledgment (optional)

**How to use:**
1. Fill in employee information
2. Verify training completed before signing
3. Have employee sign (digital or physical)
4. Store in compliance folder > personnel > confidentiality-agreements

---

### 3. [Training Tracker Template](./TRAINING_TRACKER_TEMPLATE.md)
Guide for creating and maintaining FERPA training records.

**Use this for:**
- Tracking all employee training
- Monitoring training expiration dates
- Audit readiness

**How to use:**
1. Create spreadsheet based on template columns
2. Store in compliance folder > personnel > training-records
3. Update as training is completed
4. Review monthly for upcoming expirations

---

## Using These Templates

### Step 1: Review and Customize
- Read through each template
- Customize company-specific information:
  - Replace `[Company Name]` with your company name
  - Add contact information
  - Adjust policies to match your procedures
  - Add company logo if desired

### Step 2: Legal Review
- **Recommended:** Have legal counsel review before first use
- Ensure compliance with:
  - FERPA requirements
  - State employment laws
  - Industry regulations
  - Your university partner agreements

### Step 3: Implement
- Train HR/Security team on procedures
- Create process for obtaining signatures
- Set up compliance folder storage
- Create tracking systems

### Step 4: Maintain
- Review annually
- Update when regulations change
- Update when company policies change
- Version control updated templates

## Checklist for New Employee Onboarding

Use this checklist to ensure all personnel security requirements are met:

- [ ] **Day 1:**
  - [ ] Employee receives AUP and Confidentiality Agreement
  - [ ] Employee enrolled in FERPA training (ED.gov)
  - [ ] Background check initiated (for Tier 2+ access)

- [ ] **Before Data Access:**
  - [ ] FERPA training completed (certificate on file)
  - [ ] Background check completed (for Tier 2+)
  - [ ] Confidentiality Agreement signed
  - [ ] AUP acknowledged and signed
  - [ ] Access tier determined based on role
  - [ ] All documents filed in compliance folder

- [ ] **Access Provisioning:**
  - [ ] Manager submits access request
  - [ ] Security Officer reviews and approves
  - [ ] IT provisions access at approved tier
  - [ ] Access logged in access tracker
  - [ ] Employee notified of access tier and limitations

- [ ] **First Week:**
  - [ ] Security awareness training completed
  - [ ] System-specific training completed
  - [ ] Incident reporting procedures reviewed

## Checklist for Annual Compliance

Use this for annual reviews:

- [ ] **Training:**
  - [ ] All active employees completed annual FERPA refresher
  - [ ] Certificates collected and filed
  - [ ] Training tracker updated
  - [ ] Overdue training identified and remediated

- [ ] **Access Review:**
  - [ ] Quarterly access reviews completed
  - [ ] Access tiers match current roles
  - [ ] Unnecessary access revoked
  - [ ] Access logs reviewed for anomalies

- [ ] **Policy Updates:**
  - [ ] Templates reviewed for accuracy
  - [ ] Policies updated if regulations changed
  - [ ] Employees notified of policy changes
  - [ ] Re-acknowledgment obtained if significant changes

- [ ] **Audit Prep:**
  - [ ] Compliance folder organized
  - [ ] All required documents present
  - [ ] Backup integrity verified
  - [ ] Access controls reviewed
